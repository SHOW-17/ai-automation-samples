# Claude Codeで業務自動化を始める3ステップ — 非エンジニアでも今日からできる

「業務を自動化したいけど、プログラミングができないから無理」——そう思っていませんか？2026年現在、AIの進化でこの常識は完全に変わりました。**Claude Code**という対話型のAI開発ツールを使えば、「やりたいこと」を日本語で伝えるだけで、業務自動化のスクリプトが手元に生まれます。本記事では、非エンジニアの方が最初の自動化を完成させるまでの3ステップを、つまずきポイントとあわせて解説します。

## ステップ1: Claude Codeの環境を整える

### Claude Codeとは何か

Claude CodeはAnthropic社が提供する**ターミナル上で動くAIコーディングアシスタント**です。Cursorのようなエディタ統合型と違い、シェルから対話的にコードを書かせ、ファイルを編集させ、コマンドを実行させることができます。「Excelの集計を自動化して」と日本語で頼めば、必要なPythonスクリプトを生成して、実際に動くところまで持っていってくれます。

従来のChatGPTとの最大の違いは、**ファイルシステムやコマンド実行に直接アクセスできる**こと。コードを生成するだけでなく、生成したコードをその場で実行・修正してくれるため、非エンジニアでも「動く成果物」が手に入ります。

### インストールと初期設定

Claude Codeを使うには次の3つが必要です。

1. **Claude APIアカウント**（[claude.com](https://claude.com)で発行、月$20-100程度）
2. **ターミナル**（Mac標準のTerminal、WindowsならWSLかGit Bash）
3. **Node.js v18以上**（公式サイトから5分でインストール可能）

これらが揃ったら、ターミナルで `npm install -g @anthropic-ai/claude-code` と打つだけ。あとは作業フォルダで `claude` と入力すれば、対話が始まります。最初の起動時にAPIキーを聞かれるので、貼り付ければ準備完了です。

### 最初のハードルを越えるコツ

非エンジニアがつまずく最大のポイントは「ターミナルの黒い画面に怯える」ことです。これは慣れの問題なので、最初は怖がらず、Claude Codeに「○○のやり方を教えて」と聞きながら進めてください。Claude自身が手取り足取り教えてくれます。インストールで30分悩むより、Claudeに直接聞いた方が早いことの方が多いです。

## ステップ2: 最初の自動化を1つ完成させる

### 何から始めるべきか

最初の自動化は**「自分が毎週やっている、地味で繰り返しの作業」**を選んでください。例えば次のようなものです。

- 毎週月曜の朝、メールから売上数字を抜き出してスプレッドシートに転記する
- 複数のサイトから情報を集めてExcelにまとめる
- 領収書のフォルダを月別に振り分ける

ここで大事なのは**スコープを絞ること**です。「業務全体を自動化したい」と頼むのではなく、「先週の月曜にやったあの作業だけを自動化したい」とピンポイントで指定します。30分以内に完成するレベルの粒度が理想です。

### Claude Codeへの依頼の仕方

依頼するときは、次の4点を伝えると精度が上がります。

1. **入力**: 何を渡すか（ファイル名・場所・形式）
2. **処理**: 何をしてほしいか（具体的な変換ルール）
3. **出力**: どこに何を出すか（保存先・ファイル名）
4. **環境**: 自分のPCのOSと使えるツール

例えばこう書きます。「`~/Documents/sales/` フォルダにあるCSVファイル全てを読み込んで、店舗ごとに月次合計した結果を `summary.xlsx` に出力するPythonスクリプトを作って。Macで実行する」。これだけでClaudeは必要なライブラリのインストールから動作確認まで自動で進めてくれます。

### 動かして・直して・覚える

最初から完璧なコードは出ません。Claude Codeで動かしてエラーが出たら、エラーメッセージをそのままClaudeに見せて「直して」と頼むだけでOKです。これを2-3往復すれば動くものになります。**「自分でコードを読めるようになる必要はない」**と割り切ってください。動けば成果、それで十分です。

## ステップ3: 自動化を運用に乗せる

### 定期実行の仕組みを作る

スクリプトが完成したら、次は「自動で動く」状態にします。Macなら `cron` か `launchd`、Windowsなら「タスクスケジューラ」を使えば、毎日決まった時間に自動実行できます。これもClaude Codeに「このスクリプトを毎日朝9時に実行する設定を作って」と頼めば、設定手順を案内してくれます。

最初は手動で1週間ほど動かして、結果に問題がないか確認してから自動化に移すのがおすすめです。いきなり完全自動にすると、エラーに気づかず誤った結果を使い続けてしまうリスクがあります。

### 失敗から学ぶ3つの教訓

実際に運用を始めると、想定外のことが必ず起きます。よくある失敗と対処法を3つ紹介します。

**1. 入力データの形が変わって動かなくなる** — CSVのカラム名が変更された、ファイル名のルールが変わったなど。Claudeに「異常を検知してメール通知する処理を追加して」と頼んでおくと安全です。

**2. APIの料金が想定より高くなる** — 大量データを毎日AIに投げると月数千円かかることがあります。月初に予算を決め、上限が近づいたら止まる仕組みを入れておきましょう。

**3. 自動化したのに誰も成果を見ない** — せっかく作っても誰も使わなければ意味がありません。出力先をSlackやメールにして、結果が能動的に届く形にすると活用されやすくなります。

### 次の一歩を踏み出す

最初の1つが動いたら、自然と「次はあれを自動化したい」というアイデアが湧いてきます。**1ヶ月で3つ自動化する**を目標にすると、半年後にはまったく違う働き方になっているはずです。Claude Codeは月$20から始められるので、節約できる時間を考えれば投資としては破格です。

## まとめ

Claude Codeを使った業務自動化は、3ステップで始められます。**(1)環境を整える、(2)1つだけ自動化する、(3)運用に乗せる**。プログラミングの知識はほぼ不要で、必要なのは「自分の業務を観察する目」と「Claudeに丁寧に頼む姿勢」だけです。今日の業務の中から、1つだけ自動化候補を選んで、明日Claude Codeにお願いしてみてください。1週間後の自分は、確実に楽になっています。
