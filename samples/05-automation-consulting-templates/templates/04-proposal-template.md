## 1. 現状の整理

### 1.1 ヒアリング内容のサマリ
（30分の相談で伺った内容を3〜5行で要約）

> 月次売上集計、経費精算入力、メール仕分けの3業務において、合計月間 ○時間 の手作業が発生している。

## 2. ご提案

### 2.1 推奨アプローチ

**Phase 1（1ヶ月目）: 月次売上集計の自動化**
- 効果が確実に見える
- 技術難度が低く、初期投資を最小化できる
- 経営層への成功事例として説明しやすい

**Phase 2（2ヶ月目）: メール仕分けの半自動化**
- 既存ツール（Outlook/Gmail）の標準機能で7割対応可
- AIが必要なのは「文脈で判断が必要な3割」のみ

**Phase 3（3ヶ月目以降）: 領収書OCRへの取り組み**
- 既存経費精算システムとの連携要否を要確認
- 連携可能ならROI高、不可なら見送りも検討

### 2.2 推奨ツール構成

```
[業務データ] → [Python/GASスクリプト] → [既存ツール (Excel/Slack/Outlook)]
                       ↑
              [Claude Code で開発]
                       ↑
              [月次レビュー → 改善]
```

### 2.3 自社実装 vs 外注の判断

| 項目 | 自社実装 | 外注（弊社） |
|---|---|---|
| 初期コスト | ChatGPT月額 + 学習時間 | 30,000円〜（1業務） |
| 完成までの期間 | 2〜4週間 | 1週間 |
| 保守 | 自社で対応可能 | 月額3,300円〜 |
| 推奨ケース | 学習意欲あり / 軽微な業務 | 即効性重視 / 業務影響大 |

→ 月次売上集計は**業務影響が大きいため外注を推奨**。
→ メール仕分けは**自社で取り組める難度なので学習機会としても推奨**。

## 4. リスクと対策

| リスク | 対策 |
|---|---|
| データ形式の変更で動かなくなる | 異常検知 + メール通知を組み込み |
| API利用料の予期せぬ増加 | 月次予算の上限設定 |
| 担当者が変わったときの引継ぎ | ドキュメント・操作動画を納品物に含める |
| 自動化に頼りすぎて検算しなくなる | 月次で必ず一度は人間の目視確認をルール化 |
