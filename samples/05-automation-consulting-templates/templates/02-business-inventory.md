# 業務棚卸しシート

> 自動化候補を客観的に評価するためのシートです。
> 30分の相談の中で、一緒に埋めていきます。

---

## 棚卸しの基本フォーマット

各業務について、以下の項目を埋めます。

| 項目 | 内容 |
|---|---|
| 業務名 | （短く具体的に） |
| 概要 | 1〜2行で説明 |
| 頻度 | 毎日 / 週N回 / 月N回 / 四半期 |
| 1回あたり時間 | 分単位 |
| 月間累計時間 | 頻度 × 時間 |
| 関わる人数 | 自分のみ / 同僚N人 / 全社 |
| ミスの起こりやすさ | 高 / 中 / 低 |
| ミスの影響度 | 高（売上影響） / 中 / 低 |
| 自動化の技術難度 | 高 / 中 / 低 |
| 効果スコア | 月間累計 × ミス頻度 × 影響度 |
| 候補ツール | Python / GAS / Zapier / RPA / Claude Code |
| 自動化アイデア | どう自動化するかのざっくり構想 |

## 効果スコアの算出ロジック（簡易版）

```
効果スコア = 月間累計時間(分) × 自動化可能率 × 戦略的重要度

自動化可能率: 0.3 / 0.6 / 0.9 のいずれか
戦略的重要度: 0.5 / 1.0 / 1.5 のいずれか
```

**自動化可能率の目安**:
- 0.3: 一部のみ自動化可、判断や手動工程が残る
- 0.6: 6〜7割自動化可
- 0.9: ほぼ完全自動化可

**戦略的重要度の目安**:
- 0.5: 効率化のみ
- 1.0: 効率化 + ミス削減
- 1.5: 上記 + 経営判断スピードに影響

## 自動化候補の優先度マトリクス

| | 効果大 | 効果小 |
|---|---|---|
| **難度低** | 🥇 最優先（即着手） | 🥈 余裕あれば |
| **難度高** | 🥉 計画的に取り組む | 🚫 やらない |

「難度低 × 効果大」の業務から手を付けるのが鉄則です。

## 相談後のアウトプット

このシートをもとに、以下を提案書にまとめてお渡しします:

1. 自動化候補の優先順位リスト（最大10件）
2. 推奨の最初の3件と、それぞれの実装イメージ
3. 自社実装 vs 外注の判断基準
4. 概算コスト（自社実装時の工数 / 外注時の見積）
